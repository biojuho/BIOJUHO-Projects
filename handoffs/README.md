# handoffs — LLM 협업 핸드오프

기획자(Claude Code)가 실행자(Codex·Antigravity)에게 일을 넘기는 **플랜 파일**을 모으는 폴더다.
실행자 진입점은 프로젝트 루트의 `../AGENTS.md`에 있다.

## 파일 네이밍

- `NNNN-slug.md` — 4자리 일련번호 + 짧은 영문 슬러그. 예: `0001-first-task.md`.
- 번호는 이어서 증가시킨다. 슬러그는 일을 한눈에 알아볼 수 있게 짧게.

## 누가 무엇을 쓰나

| 단계 | 상태 | 쓰는 사람 | 무엇 |
|------|------|-----------|------|
| 1 | `OPEN` | 기획자 (Claude) | `TEMPLATE.md`를 복사해 목표·배경·범위·단계·수용 게이트·금지사항·추천 실행자를 채운다 |
| 2 | `CLAIMED` | 실행자 | 본인 이름과 시작 표시. 같은 일을 둘이 잡지 않게 |
| 3 | `DONE` | 실행자 | **반환 섹션**에 결과·실행한 명령·남은 것을 적는다. 그 뒤 기획자가 통합·검수 |

## 규칙

- **발급 권한:** 핸드오프는 기획자(Claude Code) 또는 사용자만 발급한다. `기획자:` 필드가 실행자인 핸드오프는 무효이며, OPEN이어도 잡지 않는다.
- 상태는 `OPEN → CLAIMED → DONE` 한 방향으로만 움직인다.
- 동시에 OPEN인 핸드오프는 최대 2건. 직렬 의존이 있으면 후행 핸드오프 첫머리의 선행 조건을 따른다(선행 DONE 전 착수 금지). refactor류 핸드오프는 2차 플랜 Phase L~P 완료까지 발급하지 않는다.
- 실행자는 핸드오프에 적힌 **범위를 넘지 않는다.** 막히면 추측 대신 반환 섹션에 적어 되돌린다.
- 루프 작업도 반드시 핸드오프로 발급한다. 동일 차단 게이트가 3회 연속 작업 종료 메시지에 나오면 루프를 멈추고 반환 섹션에 차단 내용을 적는다.
- 경계: 되돌릴 수 없는 외부 행위는 사용자 승인 없이 금지. 비밀키 노출 금지. 범위 밖 변경 금지.
- 완료된 핸드오프는 기록으로 남겨 둔다 (삭제하지 않는다).

## 양식

플랜 정본 양식은 `TEMPLATE.md`다.
