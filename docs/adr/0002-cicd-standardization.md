# ADR-0002: CI/CD 파이프라인 표준화 및 자동 QA 도입

- **상태**: 승인됨
- **날짜**: 2026-03-08
- **의사결정자**: biojuho

## 컨텍스트

7개 프로젝트에 개별 CI 워크플로가 존재하지만, 보안 스캔/의존성 감사/QA 자동화가
통합 게이트로 묶여있지 않았습니다. 테스트 커버리지 측정도 없어 품질 추적이 불가능했습니다.

## 결정

1. **`security-quality-gate.yml`**: 전체 PR에 적용되는 통합 게이트
   - gitleaks (시크릿 누출 차단)
   - pip-audit / npm audit (의존성 취약점)
   - bandit (Python 보안 분석)
   - 시크릿 스캔 실패 시 PR 머지 차단

2. **`test-coverage.yml`**: Python 프로젝트 커버리지 리포트
   - 초기 목표: 60% (단계적 상향)
   - XML/HTML 리포트 + GitHub Summary

3. **`docker-compose.dev.yml`**: 로컬 통합 개발 환경
   - 전 프로젝트를 한 명령어로 기동

## 대안

1. **개별 프로젝트에 보안 스캔 각각 추가**: 중복 관리, 표준 일관성 부족
2. **SonarQube 도입**: 인프라 비용 과다, 현 규모에 불필요
3. **Codecov 서비스 연동**: 무료 티어 제한, 셀프호스팅 리포트로 충분

## 결과

- PR 머지 전 보안/품질 게이트가 자동 적용됨
- 커버리지 추이를 PR 단위로 추적 가능
- 로컬 개발 환경 통일로 "내 PC에서는 되는데" 문제 예방
