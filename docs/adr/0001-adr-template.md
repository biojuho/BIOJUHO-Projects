# ADR-0001: ADR 템플릿

- **상태**: 승인됨
- **날짜**: 2026-03-08
- **의사결정자**: biojuho

## 컨텍스트

프로젝트가 성장하면서 기술 결정의 배경과 근거를 기록할 체계가 필요합니다.
ADR(Architecture Decision Record)은 각 결정의 "왜"를 문서화하여
미래의 팀원(또는 미래의 자신)이 컨텍스트를 빠르게 파악할 수 있게 합니다.

## 결정

모든 주요 기술 결정은 `docs/adr/` 에 ADR 문서로 기록합니다.

### ADR 작성 기준

- 새로운 라이브러리/프레임워크 도입
- 아키텍처 패턴 변경 (예: 모놀리스 → 마이크로서비스)
- 데이터 저장 방식 변경
- API 설계 방식 변경
- 주요 인프라 결정

## 대안

1. **Notion에만 기록**: 코드와 분리되어 동기화 부족
2. **코드 주석만 사용**: 전체 맥락 파악 어려움
3. **README에 통합**: 파일이 너무 길어짐

## 결과

- 모든 기술 결정에 대해 추적 가능한 이력이 생김
- 새 팀원 온보딩 시 "왜 이렇게 했는지" 빠르게 파악 가능
- 코드 리뷰 시 ADR 참조로 논의 효율화
