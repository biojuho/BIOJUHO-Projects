# AutoResearch: GitHub Agent Platform Source Expansion

## Objective

Expand the GitHub modernization radar from the existing `17` sources to cover
additional current agent-platform, TypeScript AI app, continuous-agent, and
visual workflow-builder projects that map to the user's launch-hardening and
self-improving AutoResearch objective.

## Sources Checked

- `microsoft/agent-framework`: current Microsoft agent framework direction for
  multi-agent workflows, MCP/A2A interop, and observable production execution.
- `agno-agi/agno`: full-stack agent platform patterns for agents, teams,
  workflows, memory, knowledge, tools, and observability.
- `vercel/ai`: TypeScript AI app toolkit patterns for tool calls, provider
  abstraction, streaming UI, and AI Gateway boundaries.
- `Significant-Gravitas/AutoGPT`: continuous agent platform patterns,
  marketplace packaging, and hosted autonomous execution boundaries.
- `FlowiseAI/Flowise`: visual low-code AI agent and workflow builder patterns
  for operator-facing flow inspection and execution separation.

## A/B Contract

- Baseline: keep the radar at `17` repositories and rely on already-mapped
  agent workflow, browser automation, typed-eval, and human-checkpoint sources.
- Variant: add the five primary GitHub repositories above, map each to existing
  local evidence, regenerate the radar report, refresh live GitHub source
  metadata, and require the expanded count in the completion contract.
- Primary KPI: source coverage increases from `17` to `22` while every local
  evidence path validates and live source freshness returns `22/22` pass.
- Guardrails: do not claim hosted runtimes, visual builders, persistent browser
  sessions, Vercel AI runtime migration, Microsoft Agent Framework deployment,
  or AutoGPT-style unattended autonomy as locally adopted.
- Decision rule: adopt only if focused radar/source tests pass, live GitHub
  freshness passes for all `22` sources, pre-push-equivalent tests and runtime
  probes pass, and the completion audit stays valid.
