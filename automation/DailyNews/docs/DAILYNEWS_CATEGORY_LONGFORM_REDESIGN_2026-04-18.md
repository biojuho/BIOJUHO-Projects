# DailyNews Category Longform Redesign

Date: 2026-04-18

## Why This Change

DailyNews의 X 롱폼은 단일 기사형 글쓰기보다, 카테고리별로 끊어 읽히는 브리핑 카드 포맷에 더 가깝다.  
특히 카툰형 이미지를 함께 운영하는 만큼 텍스트가 비유를 반복하기보다 팩트와 인과, 독자 관점을 담당해야 한다.

이번 개편은 아래 피드백을 실행 규칙으로 고정하는 데 목적이 있다.

- 카테고리별 자기완결적 구조
- 메인 1개 + 서브 2~3개 계층화
- 카툰 이미지와 텍스트의 역할 분업
- 카테고리별 고유 해석 프레임
- 공유 가능한 결론 문장 강화

## Applied Changes

### 1. Insight prompt redesign

`insights/generator.py`의 인사이트 생성 프롬프트에 다음 규칙을 추가했다.

- 결과물은 카테고리형 X 롱폼 카드의 재료임을 명시
- 카툰 이미지는 비유/인상, 텍스트는 팩트/인과/함의를 담당한다고 명시
- 첫 번째 인사이트는 메인, 나머지는 서브로 생성
- 각 인사이트에 `hook`, `summary_fact`, `why_now`, `one_line_takeaway`, `closing_line`, `visual_brief`를 함께 생성

### 2. Category-aware X longform formatter

`x_long_form` 출력 구조를 아래로 표준화했다.

1. 카테고리 헤더
2. 오늘 한 줄
3. 메인 아이템 1개
4. 서브 아이템 2~3개
5. 오늘의 결론
6. 답해야 할 질문

각 아이템은 다음 네 줄 안에서 정리된다.

- 핵심 사실
- 왜 중요하냐면
- 내 시각
- 체크포인트

### 3. Category framing

카테고리별 마감 질문을 고정했다.

- 정치: 누가 권한을 얻고, 누가 제약받는가?
- 국내 경제: 이 정책의 비용은 누가 내고, 수혜는 누가 받는가?
- 글로벌 경제: 한국 자산·환율·물가에 어떤 파급이 오는가?
- AI: 이 기술이 실제 업무를 얼마나 대체하거나 보조하는가?
- Tech: 플랫폼 힘의 균형이 어디로 이동하고, 지금 무엇을 배워야 하나?
- Crypto: 시장이 착각하는 건 무엇이고, 제도권 편입은 빨라지는가?

### 4. Shared prompt wiring

`brain_adapter.py`가 `config/x_longform_prompt.json`의 규칙을 실제 프롬프트에 주입하도록 연결했다.  
이제 브레인 분석 경로에서도 동일한 롱폼 운영 규칙을 참고한다.

## Editorial Rules

- 이미지가 이미 비유를 수행하면 텍스트는 숫자와 맥락으로 들어간다.
- 한 뉴스에서 너무 많은 메시지를 동시에 말하지 않는다.
- 결론은 추상 명사가 아니라 공유 가능한 문장으로 끝낸다.
- 카테고리별 톤은 유지하되, 반복 포맷으로 학습 가능한 리듬을 만든다.

## Expected Outcome

- X 체류 시간 증가: 도입부와 오늘 한 줄의 즉시성 강화
- 시리즈 인지 강화: 카테고리별 반복 포맷 정착
- 이미지 제작 효율 증가: `visual_brief`를 통해 카툰 방향 분리
- 브랜드 선명도 강화: 카테고리별 질문 프레임 축적
